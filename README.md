# Corporate Research Agent（企業研究エージェント）

就活の企業研究を自動化する Claude Code プロジェクト。

企業名を渡すと、役割の異なる4体の調査エージェントが並列でWeb調査を行い、結果を統合 → 独立した検証エージェントが出典に当たって裏取り → Notionのデータベースにデザインされたページとして出力します。

## 使い方

このフォルダで Claude Code を起動し、次のように入力するだけです。

```
/research ソニーグループ
```

所要時間は1社あたり数分〜10分程度。完了するとNotionページのURLと注目ポイントが報告されます。

### 前提条件

- Claude Code がインストールされていること
- Notion コネクタ（MCP）が接続されていること（`/mcp` で接続状況を確認できます）
- 出力先のNotion DB「企業研究エージェント結果」にコネクタからアクセスできること

## 仕組み

```
/research 企業名
   │
   ▼
オーケストレーター（メインセッション）
   ├─ Step 1  企業の特定（正式名称・証券コード・公式サイト・業界）
   ├─ Step 2  4体の調査エージェントを並列起動
   │            ├ business-analyst        理念・Purpose・事業・ビジネスモデル
   │            ├ finance-analyst         業績推移・財務・直近決算
   │            ├ hr-culture-researcher   求める人材像・選考フロー・年収・社風
   │            └ strategy-news-researcher 直近ニュース・競合比較・面接対策
   ├─ Step 3  統合（数値の矛盾はIR一次情報を優先、ハイライト用キーワード確定）
   ├─ Step 4  fact-checker が出典URLを実際に開いて数値・引用・日付を裏取り
   ├─ Step 5  Notionにページ作成（デザインテンプレートに従う）
   └─ Step 6  output/ にMarkdownを保存し、ユーザーに報告
```

### ファイル構成

| パス | 役割 |
|---|---|
| `.claude/skills/research/SKILL.md` | `/research` コマンド本体（上記フローの指示書） |
| `.claude/skills/research/notion-template.md` | Notionページのデザインテンプレート |
| `.claude/agents/*.md` | エージェント定義（調査4体 + 検証1体） |
| `CLAUDE.md` | プロジェクト共通ルールとNotion DBのID |
| `output/` | 生成されたレポートのローカル保存先（バックアップ兼履歴） |

### 信頼性のための設計

- すべての事実に**出典URLと時点**を必須化（出典のない記述は統合時に削除）
- 調査エージェントとは独立した **fact-checker** が、重要な数値・原文引用・日付について出典を実際に開いて照合
- 検証不能だった項目は削除するか *（未確認）* と明記
- 口コミ由来の情報は「傾向」として公式情報と区別して記載

## 使われるClaudeモデルについて

サブエージェントは、**デフォルトではそのセッションで使っているモデルを継承**します（`/model` で切り替えた場合はそれに従う）。

エージェントごとにモデルを固定したい場合は、`.claude/agents/*.md` の frontmatter に `model:` を追加します。

```yaml
---
name: finance-analyst
description: ...
tools: WebSearch, WebFetch
model: sonnet   # 例: 調査はSonnetでコストを抑える（省略時はセッションのモデルを継承）
---
```

調査の網羅性を重視するなら継承のまま、コストを抑えたいなら調査4体を `sonnet` にして fact-checker と統合（メインセッション）だけ上位モデルにする構成がおすすめです。

## エージェントのやり取りを見るには

- **実行中**: 各エージェントの動作（実行中のツール・経過）はターミナル上に折りたたみ表示され、リアルタイムで進捗が見えます
- **詳細表示**: `Ctrl+O` でトランスクリプト表示に切り替えると、オーケストレーターが各エージェントに渡したプロンプトと、各エージェントの最終報告の全文を確認できます
- **実行後**: セッションログ全体（サブエージェントの全ツール呼び出し含む）は `~/.claude/projects/<このプロジェクト>/` 配下のJSONLファイルに保存されています。また、統合後の最終レポートは `output/` にMarkdownとして残ります

なお「エージェント同士の直接対話」ではなく、**並列調査 → オーケストレーターが統合 → 検証AIが裏取り**という構成です。各エージェントへの指示と報告はすべてオーケストレーター経由なので、やり取りの全体像はトランスクリプトで追跡できます。

## カスタマイズ

- **調査観点を変える**: `.claude/agents/` の該当ファイルの調査項目を編集
- **ページデザインを変える**: `.claude/skills/research/notion-template.md` を編集
- **出力先DBを変える**: `CLAUDE.md` と `SKILL.md` 内の `data_source_id` を差し替え

## 注意事項

- Web検索を多用するためトークン消費は大きめです（1社あたりの実行コストを意識して使ってください）
- 検証AIを通していますが、面接で使う重要な数値（売上高・初任給など）は必ずページ内の出典リンクから一次情報を最終確認してください
