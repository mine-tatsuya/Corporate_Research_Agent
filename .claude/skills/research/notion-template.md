# Notionページ デザインテンプレート

`/research` で作成するNotionページは、以下のデザインルールと骨格に従うこと。
Notion-flavored Markdown仕様（callout / toggle / columns / 色付け）を最大限活用し、「読みたくなる」ページにする。

## デザインルール

1. **ハイライト**: 企業が重要視しているキーワード（Step 3で確定した3〜5個）は、本文中に登場するたびに `<span color="yellow_bg">キーワード</span>` でハイライトする
2. **強調**: 面接で使える重要な事実・数値は **太字**。特に決定的なもの（その企業らしさを示す一点）は `<span color="red">赤文字</span>`
3. **色の使い分け**: 強み・ポジティブ情報 = green系 / 弱み・リスク = red系 / 理念・Purpose = blue系 / 人材・採用 = yellow〜orange系
4. **原文引用**: 理念・Purpose・採用メッセージは引用ブロック（>）で原文のまま
5. **トグル**: 補足的な情報（出典一覧、想定質問の回答例、口コミ詳細）はトグルに格納してページをすっきり保つ
6. **表**: 必ず `header-row="true"` を付ける。比較表では自社の列に色を付ける
7. **出典**: 各セクション末尾にそのセクションの出典をトグルでまとめる。本文中の重要数値には `（出典: [媒体名](URL)、2026年3月期）` のように時点付きで付記

## ページ骨格

```
<callout icon="📌" color="blue_bg">
	**30秒サマリー**
	（この企業の全体像と「一言でいうと何の会社か」を3〜5行で。志望動機の核になりそうな点を太字に）
</callout>

<table_of_contents color="gray"/>

# 🧭 企業理念・Purpose {color="blue"}

> **Purpose**<br>「（原文引用）」 {color="blue"}

（ミッション・ビジョン・バリューを箇条書き。重要キーワードはハイライト）

	**この会社が大切にしていること**: （キーワード3〜5個と、それが理念・中計・採用メッセージのどこに現れるかを1〜2行で）
</callout>

# 🏢 事業・ビジネスモデル

（主力事業の説明、セグメント別売上構成の表）

	<column>

		</callout>
	</column>

		</callout>
	</column>

（中期経営計画: 計画名・期間・数値目標・重点領域）

## 💰 年収・待遇

（表: 初任給（学部/修士）/ 平均年収（有報・時点付き）/ 年代別目安（口コミベースと明記）/ 主要福利厚生）

## 🏛 社風・働き方

（公式データ: 残業・有給・離職率の表 → 口コミの傾向は「良い面/気になる面」の2カラムで）

# 📰 直近の重要ニュース

（表: 日付 / 出来事 / 就活でどう使うか。ネガティブニュースも含める）

# ⚔️ 競合比較

（比較表: 自社の列に blue 色を付ける。売上規模・強み・平均年収・社風）

<callout icon="🥇" color="green_bg">
	**なぜ競合ではなくこの会社か（差別化ポイント3つ）**
</callout>

# 🎤 面接対策

<details color="gray_bg">
<summary>**想定質問と回答のヒント**</summary>
（質問ごとに: 質問 / なぜ聞かれるか / 回答の方向性）
</details>

<details color="gray_bg">
<summary>**逆質問のネタ**</summary>
（ニュース・中計を踏まえた「調べてきた感」が出るもの3個）
</details>

# 🔗 出典・参考文献

<details>
<summary>**全出典リスト（検証済みマーク付き）**</summary>

</details>

<callout icon="🤖" color="gray_bg">
	このページはAIによる自動調査（調査日: YYYY-MM-DD）。重要な数値は出典リンクから一次情報を確認してください。
</callout>
```

## 注意

- `<empty-block/>` 以外の空行は削除されるため、空行に頼らない
- リスト項目内の子要素・callout内の子要素はタブでインデントする
- 表のセル内では Notion-flavored Markdown のみ使用（HTMLタグ不可）
- エスケープが必要な文字: `\ * ~ ` $ [ ] < > { } | ^`
