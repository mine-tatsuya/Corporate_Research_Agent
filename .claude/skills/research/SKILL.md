# 企業研究の実行手順

引数で渡された企業名について、以下の手順で企業研究を実行する。途中で各ステップの進捗をユーザーに簡潔に伝えること。

## Step 1: 企業の特定

WebSearchで以下を特定する:
- 正式名称（法人名）
- 証券コード（上場企業の場合）と上場市場
- 公式サイトURL・新卒採用サイトURL
- 業界（DBの「業界」セレクト: メーカー / IT・通信 / 商社 / 金融 / コンサル / インフラ・エネルギー / 小売・サービス / その他 のいずれか）

同名・類似名の企業がある場合は、就活文脈で最も可能性が高い企業を選び、「○○として調査します」と一言伝えてから進める。

## Step 2: 並列調査（4体の調査エージェント）

以下の4体をAgentツールで**1つのメッセージで同時に**起動する。各エージェントにはStep 1で特定した正式名称・証券コード・公式サイトURL・採用サイトURLを必ず渡す。

| エージェント | 担当 |
|---|---|
| business-analyst | 理念・Purpose・事業・ビジネスモデル・強み弱み・中計 |
| finance-analyst | 業績推移・財務健全性・直近決算・株価 |
| hr-culture-researcher | 求める人材像・選考フロー・年収・働き方・社風 |
| strategy-news-researcher | 直近ニュース・競合比較・面接対策 |

※ カスタムエージェントが subagent_type として認識されない場合は、`.claude/agents/` の該当ファイルの内容をプロンプトに含めて general-purpose エージェントで代替する。

## Step 3: 統合

- 4つの報告を統合し、レポート草稿を作る
- 数値の矛盾（売上高・年収等）があればIR一次情報を優先する
- 各エージェントが抽出した「企業が重要視しているキーワード」を突き合わせ、ページ全体で一貫してハイライトするキーワード（3〜5個）を確定する
- **すべての事実に出典URLが付いているか確認する。** 出典のない記述はこの時点で削除するか出典を補う

## Step 4: 検証（fact-checker）

- 統合した草稿を fact-checker エージェントに渡し、重要な数値・引用・日付の裏取りをさせる
- 「⚠️ 要修正」は草稿を修正、「❓ 検証不能」は削除するか *（未確認）* の注記を付ける
- 検証で大きな誤りが見つかった場合は、その分野の調査エージェントに再調査を依頼してもよい

## Step 5: Notionページの作成

同じディレクトリの `notion-template.md` のデザインテンプレートに**厳密に**従い、Notion-flavored Markdownでページを作成する。

### 5-1. ページ作成（`notion-create-pages`）

**必ず2ステップに分けて行う。全文を一度に送ると失敗するため禁止。**

**ステップA: 最小ページ作成**

`notion-create-pages` で本文は短いプレースホルダー（例: `調査中`）だけにして、プロパティとアイコンだけを設定する。

- 親: `data_source_id` = `37cedb10-c049-8056-ac85-000bc042bc44`
- プロパティ（パラメータに渡す値）:
  - `企業名`: 正式名称
  - `業界`: Step 1の結果（メーカー / IT・通信 / 商社 / 金融 / コンサル / インフラ・エネルギー / 小売・サービス / その他）
  - `証券コード`: コードと市場名（例: `7203（東証プライム）`）
  - `公式サイト`: URL文字列
  - `date:調査日:start`: 今日の日付（`YYYY-MM-DD` 形式）
  - `選考状況`: `未応募`（ユーザーが既に選考中と言っている場合は該当ステータス）
  - `一言サマリー`: 40字以内
- アイコン: 企業に合う絵文字（emoji文字を直接指定）

> **⚠️ 文字化け・パラメータ未定義エラーの防止**
> JSON パラメータ内の日本語文字列は必ず **Unicode エスケープ**で渡す。
> 例: `トヨタ自動車株式会社` → `トヨタ自動車株式会社`
> 例: `メーカー` → `メーカー`
> 例: `未応募` → `未応募`
> これを怠ると `pages expected array, received undefined` エラーが発生する。

**ステップB: 本文の投入**（ページ作成成功を確認してから実行）

`notion-update-page` の `replace_content` コマンドで全文を投入する。

```
command: replace_content
page_id: ステップAで取得したページID
new_str: （notion-template.md に従った全文）
allow_deleting_content: true
```

### 5-2. プロパティ設定まとめ

| プロパティ | 値 | 備考 |
|---|---|---|
| 企業名 | 正式名称 | Unicode エスケープ必須 |
| 業界 | セレクト値 | Unicode エスケープ必須 |
| 証券コード | コード（市場名） | Unicode エスケープ必須 |
| 公式サイト | URL | ASCII のみなのでエスケープ不要 |
| date:調査日:start | YYYY-MM-DD | ASCII |
| 選考状況 | 未応募 等 | Unicode エスケープ必須 |
| 志望度 | 空欄 | ユーザーが自分で設定 |
| 一言サマリー | 40字以内 | Unicode エスケープ必須 |

## Step 6: ローカル保存と報告

- 同内容のMarkdownを `output/YYYY-MM-DD_企業名.md` に保存する（検証結果サマリーも末尾に含める）
- ユーザーへの最終報告: NotionページのURL、特に注目すべき発見3点、検証で修正・削除した項目があればその概要

## エラー時の振る舞い

- Notion MCPが使えない場合: ローカル保存のみ行い、その旨を伝える
- 調査情報が極端に少ない企業（非上場ベンチャー等）: わかった範囲でページを作り、情報の確度が低いことをページ冒頭に明記する
